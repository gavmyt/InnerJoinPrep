Inner Join

Inner Join is an SQL command that allows you to link two different tables together whether it is to see all of the data, combine data, and more. The two different tables that you are merging need to have one category between them that is the same, as you have to set two categories equal to each other. To start off with inner join, you need to have a select command. Select acts like normal, except if you want to select a specific category you have to put "Table.Category", so within the SQL Database we've been working with a select command for two different tables would look like:

SELECT Orders.OrderDate, Shippers.ShipperName

This would return just those two categories later on once we call those two categories. The next syntax is FROM, and you will just call one of the databases that you will work with. Pulling from our example SQL database we will use Orders. This is the same as if you were only using Orders normally, so nothing changes here

Next we begin to use the inner join command. Inner Join is like one of our other power words, so we put INNER JOIN. Next, after INNER JOIN we put the other table that we want to pull in. As said earlier, we are going to use the Shippers table, so we put INNER JOIN Shippers. Next you have to put ON, which is the syntax used to link the two charts. Next, you put your two categories that are the same, with their tables in front, and an equal sign in between them. So this would look like Orders.ShipperID = Shippers.ShipperID

All together this syntax looks like:

SELECT * // or (SELECT Orders.OrderDate, Shippers.ShipperName) if you are wanting to select specific categories as mentioned earlier

FROM [Orders]

INNER JOIN Shippers ON Orders.ShipperID = Shippers.ShipperID

These are all of the components to describe inner join and how to make it work.

Data Scientists

A data scientist is someone who finds new discoveries through data analysis. By visualizing data, running algorithms to search through and interpret the data, and using that data to answer questions and solve problems, data scientists set themselves apart from a typical computer scientist. While data science and computer science do share some similarities, they are fundamentally different from each other and both answer different questions and solve different issues.

One of the things that data scientists deal with a lot more than computer scientists is, well... data! Shocker. Data scientists analyze data and sort through data. There are many uses with this data, whether they are analyzing it for trends, creating new algorithms based off of it, or presenting it in a visually pleasing manner. Computer scientists deal more with solving problems by creating the solution through computing. They are trying to solve issues with the computing tools we have.

Data scientists have many jobs. Data scientists' first job is to understand the problem at hand that needs to be solved. A data scientists' next job is data acquisition, or acquiring all of the necessary relevant data to deal with the problem at hand. After the data is gathered, the next two steps are data cleaning and data transformation. Data cleaning is the most time consuming step of the process as you have to clean up and make sure everything is spelled and formatted correctly so that the data is not incorrect. Data transformation is where you modify the data. You transform the data so you can further understand the data better. Next is exploratory data analysis (EDA), which is the most important step. EDA defines and refines the variables that will be used in the data model. You have to do this, or you could select the wrong variables and make the data model incorrectly. Next is data modeling, where through various algorithms you have to find the best model to model the data. Next is the data visualization step where you have to find the easiest way to visualize the data found in the last step.
